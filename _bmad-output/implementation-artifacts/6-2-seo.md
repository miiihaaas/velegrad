# Story 6.2: SEO — meta/OG per zapis, sitemap.xml/robots.txt, Schema.org RealEstateListing, GA4

## Opis

As a vlasnik sajta,
I want SEO optimizaciju svake stranice i nekretnine,
So that sajt bude vidljiv na pretraživačima i deljiv na društvenim mrežama.

Ovo je **DRUGA priča Epika 6 (Faza 6 + 7 — Multilingual, SEO, performanse, deploy)** i ona **realizuje NFR-3 (SEO)**. Nadograđuje se NAD Story 6.1 (dvojezičnost je već aktivna: `i18n_patterns(prefix_default_language=False)`, `LocaleMiddleware`, `{% loc %}` tag, `.po`/`.mo` prevodi) — pa SEO sloj mora biti **jezik-svestan** gde je to relevantno (meta/OG/title se renderuju na aktivnom jeziku; sitemap pokriva javne rute).

Trenutno stanje (nasleđeno — POTVRĐENO čitanjem koda):
- **Modeli VEĆ imaju sva SEO polja (BEZ MIGRACIJE):** `Property.meta_title` (`properties/models.py` l.131, `max_length=70, blank=True`) + `Property.meta_description` (l.132, `TextField, blank=True`); `Page.meta_title` (`pages/models.py` l.20) + `Page.meta_description` (l.21); `SiteSettings.google_analytics_id` (`core/models.py` l.75), `SiteSettings.seo_default_title` (l.79), `SiteSettings.seo_default_description` (l.80). **6.2 NE uvodi migraciju** — sva polja postoje od Epika 1 (§5.2).
- **`base.html` `<head>` VEĆ ima sve override blokove (Story 2.1 ih ostavila za Epik 6):** `{% block title %}` (l.12), `{% block meta_description %}` sa **hardkodovanim srpskim defaultom** (l.14–16), `{% block og %}` sa hardkodovanim `og:title`/`og:description`/`og:type`/`og:image` (l.18–24), `{% block schema %}{% endblock %}` (l.26 — „JSON-LD Schema.org — spremno za Epik 6"), `{% block ga4 %}{% endblock %}` (l.27 — „GA4 placeholder — TODO: dodati kada stigne GA4 measurement ID"). 6.2 ih **PUNI** dinamičnim, per-zapis i per-jezik sadržajem.
- **`django.contrib.sites` + `django.contrib.sitemaps` NISU u `INSTALLED_APPS`** (`config/settings/base.py` l.53–78) — 6.2 ih DODAJE (+ `SITE_ID = 1`). `core/sitemaps.py` **NE POSTOJI** (arh §2 l.129 ga predviđa) — 6.2 ga kreira (`Property` + `Page` sitemap). `/sitemap.xml` i `/robots.txt` rute **NE POSTOJE** u `config/urls.py` — 6.2 ih dodaje **VAN `i18n_patterns`**.
- **GA4 izvor = `SiteSettings.google_analytics_id`** (admin-editable, §5.2) — NE settings/env varijabla (arh §3 navodi `GOOGLE_ANALYTICS_ID` env, ali epics 6.2 AC4 eksplicitno traži `SiteSettings.google_analytics_id`; klijent ga unosi kroz CMS bez deploy-a — vidi Dev Notes).
- `site_settings` (SiteSettings singleton) + `LANGUAGE_CODE` (aktivni jezik) su VEĆ u SVAKOM template-u (`core.context_processors.site_settings` l.113 + `context_processors.i18n` l.110) → globalni meta/OG default i GA4 u `base.html` čitaju `site_settings.*` bez novog context processora.

**Obim ove priče = SEO SLOJ (NFR-3 / arh §4, §5):**
1. **Per-zapis meta + Open Graph:** `base.html` globalni default iz `SiteSettings.seo_default_*`; per-zapis override na Property Detail (`property.meta_title`/`meta_description` + per-property OG sa `hero_image`) i Page stranicama (`page.meta_title`/`meta_description`), jezik-svesno.
2. **`/sitemap.xml` + `/robots.txt`** preko `django.contrib.sitemaps` (`core/sitemaps.py` — `Property` aktivni + `Page` aktivni URL-ovi), rute VAN `i18n_patterns`.
3. **Schema.org `RealEstateListing` JSON-LD** na Property Detail (`{% block schema %}`).
4. **GA4** u `base.html` (`{% block ga4 %}`) preko `SiteSettings.google_analytics_id` — renderuje se SAMO kad je ID postavljen.

**STROGO van obima → ostale priče / odloženo (vidi Dev Notes — Granica):**
- **Performanse/WebP/`srcset`/lazy-load** = Story 6.3.
- **Deploy (Nginx/Gunicorn/Certbot, Mailgun SPF/DKIM)** = Story 6.4.
- **Facebook Pixel** (`SiteSettings.facebook_pixel_id` postoji u modelu) = **van MVP** (PRD §2 odloženo, arh §8). 6.2 NE renderuje FB Pixel.
- **`hreflang` alternate linkovi (SR↔EN) u `<head>`** = best-effort/opciono (nije eksplicit u epics 6.2 AC); ako se radi, mora biti bezopasno. NIJE DoD blocker za MVP (vidi Dev Notes — i18n SEO granica).
- **FR meta/sadržaj** = NE prevodi se (van MVP, kao 6.1).
- **`google_analytics_id` u settings/env** umesto SiteSettings = NE (epics 6.2 AC4 LOCKED na `SiteSettings.google_analytics_id`).
- **Migracija modela** = NE — sva `meta_*`/`seo_default_*`/`google_analytics_id` polja VEĆ postoje (§5.2, Epik 1).

- **⚠️ GA4 inline-JS bezbednost — `escapejs`, NE goli auto-escape (ISPRAVKA):** Ranija tvrdnja „auto-escape je dovoljan za inline `gtag('config','{{ id }}')`" je **NETAČNA**. Django HTML auto-escape escape-uje HTML meta-znakove (`<`/`>`/`&`/`"`/`'`), ali `gtag('config','...')` je JS-string kontekst — stray apostrof/navodnik bi probio JS-string (auto-escape `'`→`&#x27;` u HTML atributu pomaže, ali unutar `<script>` bloka HTML-escape semantika ne važi isto). ZATO inline config vrednost MORA proći `|escapejs` (ili ceo config preko `json_script`). Dodatno: goli `{% if site_settings.google_analytics_id %}` PROPUSTA whitespace-only string (`"   "` je truthy) → renderovao bi nevažeći snippet; guard MORA biti na obrezanoj vrednosti (`...google_analytics_id.strip` u template-u ili strip u context processoru) → whitespace-only se tretira kao „bez ID-a". Vidi T5/AC4 (whitespace test case).

- **Schema.org JSON-LD konstrukcija — LOCKED na `json_script` (XSS/JSON-break/Decimal):** `<script type="application/ld+json">` sadrži JSON; ručno sklapanje u template-u (sa zarezima/ručnim escape-om) poziva trailing-comma, neescape-ovan `</script>`, i `Decimal`-kao-string bug-ove. **LOCKED (direktiva, ne preferenca):** gradi Schema dict u view-u (`PropertyDetailView._build_context` → `schema_data`) i renderuj kroz Django `{{ schema_data|json_script:"schema-org" }}` (proizvodi VALIDAN JSON, auto-escape-uje `<`/`>`/`&`/`</script>`). **Ručni JSON u template-u NIJE dozvoljen.** Odvojeno od JSON-escape-a: ako `description` potiče od HTMLField (`description_sr/_en`), MORA proći `strip_tags(...)` + `Truncator(...).words(N)` U VIEW-u pre dict-a — `json_script` NE skida HTML tagove (samo ih JSON-escape-uje), pa bi tagovi ostali u tekstu.
