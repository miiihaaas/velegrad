# Story 5.2: Inquiry pipeline — čuvanje, notifikacija agentu, auto-reply kupcu, anti-spam

## Opis

As a agent (Đorđije),
I want da svaki upit sa platforme bude sačuvan, da odmah dobijem email notifikaciju, a kupac brendiran auto-reply,
So that nijedan lead nije izgubljen i kupac odmah oseti premium uslugu.

Ovo je **POSLEDNJA priča Epika 5 (Faza 5 — Private Collection i inquiry sistem)** i ona **zatvara FR28 inquiry pipeline**. Tri od četiri komponente FR28 su VEĆ isporučene u ranijim pričama: **(a) čuvanje svih tipova upita u bazu** (`create_inquiry` write-seam — 3.2 Property Detail `viewing`, 4.2 Contact `general`, 5.1 Private Collection `private_collection`) i **(d) anti-spam** (CSRF Django default + honeypot `website` + `@ratelimit` 5/h + `ip_address` capture — 4.2/5.1). **Story 5.2 dodaje JEDINI preostali deo FR28 — EMAIL PIPELINE: (b) notifikacija agentu na nov upit i (c) auto-reply kupcu (premium HTML email).**

Trenutno stanje (nasleđeno — POTVRĐENO čitanjem koda):
- `inquiries/services.py` ima `create_inquiry(*, form, property, inquiry_type, request=None)` — JEDAN reusable write-seam koji radi `form.save(commit=False)` → server-side polja (`inquiry_type`/`property`/`status="new"`/`preferred_language="sr"`/`ip_address`) → `inquiry.save()`. **Servis je VEĆ dizajniran sa email-hook tačkom za 5.2** — eksplicitni komentari: docstring l.6 „Story 5.2 hooks the email notification / auto-reply here (NOT here yet)", l.16 „NO email send here (deferred to 5.2)", l.51 (posle `inquiry.save()`) „# NO email send here (5.2)". **5.2 popunjava taj seam** — dodaje poziv u NOVI `inquiries/emails.py` posle uspešnog `inquiry.save()`.
- `inquiries/emails.py` i `templates/email/` **NE POSTOJE** — 5.2 ih kreira (arhitektura §1.5 l.106–109, §2 file tree l.141).
- `core.SiteSettings` (1.2) ima `email_inquiries = EmailField(blank=True)` (l.47 — **INTERNI inbox za agentske notifikacije; ovo je PRIMALAC notifikacije**) i `email_primary = EmailField(blank=True)` (l.46 — fallback). `SiteSettings.load()` vraća singleton. 5.2 NE menja model.
- `config/settings/` — **NEMA NIKAKVE email konfiguracije** (bez `EMAIL_BACKEND`, bez `ANYMAIL`, bez `DEFAULT_FROM_EMAIL`). `base.py` već ima `env` (django-environ) inicijalizovan i `env.read_env(.env)`. `dev.py`/`test.py`/`prod.py` ne diraju email. **5.2 dodaje email settinge po okruženju.**
- `requirements/base.txt` deklarisao `django-anymail[mailgun]` (l.28). **VERIFIKOVANO: anymail JE VEĆ instaliran u aktivnom venv-u (v15.0).** 5.2 T1 NE tvrdi da fali — samo verifikuje `import anymail` (radi) i idempotentno sinhronizuje deps (`.venv\Scripts\python.exe -m pip install -r requirements/base.txt` — bezopasno, ne instalira ništa novo ako je sve prisutno) PRE upotrebe anymail backend-a. **LOCKED:** 5.2 dodaje `'anymail'` u `INSTALLED_APPS` (`config/settings/base.py`) — dokumentovani django-anymail setup (uključuje anymail system checks + buduće status-tracking/webhooks), bezopasno u dev/test (console/locmem backend ga ignorišu). `python manage.py check` prolazi i sa i bez `'anymail'` u INSTALLED_APPS (anymail system checks se pokreću tek kad je app prisutan; Mailgun backend radi standalone), ali dodavanje je zaključana, preporučena postavka — uklanja ambiguitet.
- `.env.example` VEĆ ima `MAILGUN_API_KEY`/`MAILGUN_SENDER_DOMAIN` (l.24–25) + `DEFAULT_FROM_EMAIL` (l.26). 5.2 osvežava komentar.
- `ContactView` (4.2), `PrivateCollectionView` (5.1) i `PropertyDetailView` (3.2) svi pozivaju `create_inquiry(...)` u POST handler-u. **Pošto 5.2 hook-uje email UNUTAR `create_inquiry`, sva tri toka (viewing/general/private_collection) automatski dobijaju email JEDNOM izmenom — view-ovi se NE diraju.**

**Obim ove priče = EMAIL PIPELINE (b + c iz FR28):**
1. **Sinhronizacija zavisnosti + verifikacija** (`pip install -r requirements/base.txt`, `import anymail` radi).
2. **Email settinzi po okruženju:** `DEFAULT_FROM_EMAIL` (env-based, u `base.py`); `dev.py` → `console.EmailBackend`; `test.py` → **`locmem.EmailBackend`** (da `mail.outbox` radi u testovima); `prod.py` → `anymail.backends.mailgun.EmailBackend` + `ANYMAIL` dict (`MAILGUN_API_KEY`/`MAILGUN_SENDER_DOMAIN` iz env-a).
3. **`inquiries/emails.py`** — DVA toka: `notify_agent(inquiry)` (na `SiteSettings.email_inquiries`, fallback `email_primary`, sa svim podacima upita + linkom na admin) i `send_auto_reply(inquiry)` (brendiran HTML + plain-text na `inquiry.email`), oba preko `EmailMultiAlternatives`; orkestrator `send_inquiry_notifications(inquiry)`.
4. **`create_inquiry` hook:** posle `inquiry.save()` pozvati `send_inquiry_notifications(inquiry)` — **NON-FATAL** (mail greška se hvata/loguje, NE 500-uje formu).
5. **`templates/email/`** — HTML + `.txt` template-i za obe poruke (premium ton, brend boje).
6. **⛔ Cross-cutting:** invertovati `mail.outbox==0` asercije u **3 postojeća test fajla** (3.2/4.2/5.1) + novi testovi za email pipeline.

**STROGO van obima → kasnije priče/Epici (vidi Dev Notes — Granica):**
- **Mailgun domen SPF/DKIM/MX verifikacija + prvo produkcijsko slanje** = **Story 6.4 deploy** (arhitektura §6 l.243, epics 6.4 l.556). 5.2 SAMO wire-uje anymail backend + env varijable u `prod.py`; live deliverability je deploy-vreme.
- **Admin Upiti UI (tabela/filteri/promena statusa)** = **VEĆ urađeno 1.4** (FR25 admin deo). 5.2 pokriva SAMO email notifikaciju iz FR25.
- **GDPR retencija/anonimizacija `ip_address`** = NIJE u obimu 5.2 koda (5.1 Dev Notes ga je tagovao kao „5.2 scope", ali orkestrator obim 5.2 je email pipeline; retencija je operativna/cron stvar — tracking u 6.4/operativno, NE kodna izmena ove priče). `ip_address` se već hvata (5.1).
- **`{% loc %}` / jezički switch / `/en/`** = Epik 6 (IR #1). Email template-i renderuju SR copy kroz `{% trans %}` (aktivni jezik); višejezični email po `inquiry.preferred_language` je opciona dorada (vidi Dev Notes — preferred_language).
- **Asinhrono slanje (Celery/queue)** = VAN obima. 5.2 šalje sinhrono u request ciklusu (non-fatal, brzo dovoljno za MVP zapreminu). Async je 6.x optimizacija ako zatreba.
- **Dodatni anti-spam slojevi** = NE. Anti-spam (CSRF + honeypot + `@ratelimit` 5/h + `ip_address`) je VEĆ kompletan iz 4.2/5.1. 5.2 ga samo regresijski potvrđuje (AC5).
