# Story 5.1: Private Collection stranica i intake forma

## Opis

As a kvalifikovani posetilac,
I want diskretnu Private Collection stranicu sa intake formom,
So that zatražim pristup off-market portfoliju bez javnog izlaganja nekretnina.

Ovo je **PRVA priča Epika 5 (Faza 5 — Private Collection i inquiry sistem)**. Epici 1–4 su isporučili: CMS modele/admin (`Inquiry` model — 1.2/1.4, `SiteSettings` — 1.2/1.4), dizajn sistem + `base.html` + kompletnu CSS/JS kaskadu u `static/` (2.1), Home + `site_settings` context procesor + **Private Collection teaser koji hardkodovano linkuje `href="/private-collection/"`** (2.2), Signature listing/Property Detail (3.x), i — **ključno za 5.1** — **`InquiryForm` + `create_inquiry` write-seam + honeypot/CSRF/PRG obrazac** (3.2) i **`ContactView` (CBV) + `@method_decorator(ratelimit, name="post")` + `RATELIMIT_ENABLE=False` u test-settings + `django_ratelimit` u `INSTALLED_APPS` + `SILENCED_SYSTEM_CHECKS`** (4.2 — kompletna rate-limit infrastruktura). **Story 5.1 KREIRA javnu Private Collection stranicu (`/private-collection/`)** sa (a) **hero-om na tamnoj pozadini + tekstom objašnjenja, BEZ ijedne prikazane nekretnine/cene/adrese** (FR17 — ovo NIJE listing), i (b) **intake formom od 5 polja** (`name`, `email`, `phone`, `property_type_wanted`, `budget_range`) koja kreira `Inquiry(inquiry_type="private_collection")` u bazu (CSRF + honeypot + rate-limit + `ip_address`).

**STROGO van obima → kasnije priče/Epici (vidi Dev Notes — Granica):**

- **Centralizovani inquiry pipeline / anti-spam dorade (npr. dodatni rate-limit slojevi)** = **Story 5.2**. 5.1 koristi POSTOJEĆU rate-limit infra iz 4.2 (honeypot + `@ratelimit` 5/h + CSRF) — to je dovoljno za AC „CSRF + honeypot + rate-limit aktivni".

- **Pun SEO** (Schema.org, OpenGraph render, sitemap) = Epik 6. 5.1 SME postaviti `<title>`/meta iz statičkog `{% trans %}` ako je trivijalno, ali to NIJE kritičan AC.
- **GDPR retencija/anonimizacija `ip_address`** = 5.2 scope (`create_inquiry` već hvata `ip_address`; čuvanje je OK sada).
- **Prikaz off-market nekretnina / listing private kolekcije** = NIJE u obimu (FR4/FR17 EKSPLICITNO: Private Collection NE prikazuje nekretnine — samo hero + tekst + intake forma). NEMA query nad `Property(collection_type="private")`.
- **base.html okvir** (header/footer/lang-switcher/kaskada) = 2.1. 5.1 NE menja strukturu.
- **Produkcioni rate-limit iza Nginx-a** (deljeni cache + X-Forwarded-For) = **Story 6.4** (tracked tamo, kao u 4.2 Dev Notes). 5.1 vozi `LocMemCache` + `REMOTE_ADDR` jedno-procesni MVP (isto kao 4.2).

- [ ] **AC3 — Validan POST kreira Inquiry(inquiry_type='private_collection') u BAZU + property_type_wanted/budget_range sačuvani + CSRF + PRG (epics 5.1 AC3, FR18, NFR-5, arh §1.5).** POST `/private-collection/` sa validnim podacima (`name`/`email`/`phone`/`property_type_wanted`/`budget_range`, honeypot prazan):
  - Kreira **TAČNO jedan** `Inquiry` red sa `inquiry_type="private_collection"`, `status="new"`, `preferred_language="sr"` (server-side preko `create_inquiry`), `property=None` (Private Collection NEMA konkretnu nekretninu), `ip_address` iz `request` (REMOTE_ADDR).
  - **`property_type_wanted` i `budget_range` su SAČUVANI iz forme** (oni SU `PrivateCollectionForm` polja → `form.save(commit=False)` ih kopira na instancu pre `create_inquiry` server-side override-a; `create_inquiry` ih NE dira pa opstaju). Tampering POST (`inquiry_type=viewing`, `status=closed`, `preferred_language=en`) → sačuvan red i dalje `private_collection`/`new`/`sr`. enforce_csrf_checks POST bez tokena → 403, `count()==0`. `len(mail.outbox)==0` (email = 5.2).
