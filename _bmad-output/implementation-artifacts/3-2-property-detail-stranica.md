# Story 3.2: Property Detail stranica

## Opis

As a potencijalni kupac,
I want detaljnu luxury-brošuru stranicu nekretnine sa galerijom i direktnim kontaktom agenta,
So that se „zaljubim" u nekretninu i lako pošaljem upit za razgledanje.

Ovo je **poslednja priča Epika 3 (Faza 3 — srce kataloga)** i **PRVA priča koja PIŠE u bazu** — agent-kontakt mini forma kreira `Inquiry` red. Epici 1–2 i Story 3.1 su isporučili: CMS modele/admin (`Property` sa svim poljima, `PropertyImage`, `PropertyFeature`, `Inquiry` — 1.2/1.4), dizajn sistem + `base.html` + kompletnu CSS/JS kaskadu u `static/` (2.1), Home povezan sa bazom + `site_settings` context procesor (2.2), i Signature listing `/properties/` (`name="properties"`) sa server-side filterima (3.1). **3.1 listing kartice (i 2.2 home featured) već linkuju ka HARDKODOVANOM apsolutnom `href="/properties/<slug>/"`** — ta ruta još NE postoji. **Story 3.2 KREIRA tu detalj rutu** (`name="property-detail"`) + `PropertyDetailView` + `templates/property-detail.html` iz `docs/OpenDesignFiles/property-detail.html`, i dodaje **mini Inquiry formu** (`inquiries/forms.py` → `InquiryForm`) koja na POST kreira `Inquiry(inquiry_type='viewing', property=obj)`.

Trenutno stanje (nasleđeno):
- `properties/` app ima `models.py` (1.2), `admin.py` (1.4), `apps.py`, `preview.py` (1.4 — `can_preview`), `views.py` (3.1 — `PropertyListView`), `forms.py` (3.1 — `PropertyFilterForm`), `migrations/`. **3.2 DODAJE `PropertyDetailView` u `views.py`** (NE menja `PropertyListView`).
- `inquiries/` app ima `models.py` (1.2 — `Inquiry`), `admin.py` (1.4), `apps.py`. **NEMA `forms.py` ni `views.py`/`urls.py`** — 3.2 kreira **`inquiries/forms.py` → `InquiryForm`** (mini viewing forma). POST handling ide kroz `PropertyDetailView` (forma se submituje na istu detalj URL), pa `inquiries/views.py` NIJE nužan u 3.2.
- `config/urls.py` (3.1): `home`, `properties/` (`name="properties"`), `ADMIN_URL/`, `tinymce/` + `handler404`. **Ruta `/properties/<slug>/` NE postoji** — 3.2 je registruje (`name="property-detail"`).
- `static/css/pages/property-detail.css` i `static/js/gallery.js` su **već kopirani u 2.1** — uključuju se per-stranica preko `{% block extra_css %}`/`{% block extra_js %}`. **Leaflet NIJE u `static/`** — 3.2 ga uvodi (vidi Granica/Dev Notes).
- `properties/preview.py` `can_preview(request, obj)` (1.4): vraća `True` ako `obj.is_active` ILI ako je `request.user` autentikovan staff i `request.GET.get("preview") == "1"`. **Napravljen TAČNO za ovaj detalj view** (vidi docstring).
- `Property` model (1.2): `slug` (unique), `title`, `description_sr/en/fr` (`HTMLField`), `status`, `property_type`, `location_city/district/address`, `show_address` (bool, default False), `price`/`price_on_request`, `area_sqm`/`area_total_sqm`, `bedrooms`/`bathrooms`/`floor`/`total_floors`/`parking_spaces`, `year_built` (nullable), `hero_image` (`ImageField`), `floor_plan` (`FileField` pdf/img, blank), `virtual_tour_url` (blank), `latitude`/`longitude` (`Decimal`, nullable), `features` (M2M `PropertyFeature`), `is_active`, `collection_type`, `get_status_display`, `get_property_type_display`. `PropertyImage` (FK `property` `related_name="images"`, `image`, `caption`, `order`, `is_hero`; `Meta.ordering=["order"]`). `PropertyFeature` (`name_sr/en`, `icon`, `category`, `.localized()`). `Inquiry` (FK `property` `SET_NULL` `related_name="inquiries"`, `inquiry_type` [viewing/consultation/private_collection/general], `name`, `email`, `phone`, `message`, `preferred_language` [REQUIRED — bez default-a], `status` default `'new'`, `created_at`).

**Obim ove priče = PROPERTY DETAIL stranica + mini Inquiry(viewing) forma:**
1. `properties/views.py` → **NOVI** `PropertyDetailView` (DetailView ili funkcijski): `get_object_or_404(Property, slug=...)` + **`can_preview()` gating** (404 za neaktivnu/nepostojeću; staff preview preko `?preview=1`). GET renderuje detalj; POST obrađuje `InquiryForm` (PRG — POST-redirect-GET).
2. `inquiries/forms.py` → **NOVI** `InquiryForm(forms.ModelForm)` nad `Inquiry`: polja `name`/`email`/`phone`/`message` (sva required, email validiran) + **honeypot** skriveno polje. `inquiry_type='viewing'`, `property=obj`, `status='new'`, `preferred_language='sr'` se postavljaju **SERVER-SIDE** (NE iz POST-a).
3. `templates/property-detail.html` → `{% extends "base.html" %}`: hero galerija + thumbnail strip + lightbox, sticky info sidebar (`def-table`), storytelling opis (`description_sr|safe`), features 2-kol (`amenity-grid`), floor plan + Leaflet mapa (graceful), Agent Contact Block + mini forma. `{% block extra_css %}` `pages/property-detail.css` (+ Leaflet CSS), `{% block extra_js %}` `gallery.js` (+ Leaflet JS + map init).
4. `config/urls.py` → ruta `path("properties/<slug:slug>/", PropertyDetailView.as_view(), name="property-detail")` — **POSLE** `properties/` rute (specifičnija pre/posle ne pravi konflikt jer je `properties/` egzaktno, a ovo `properties/<slug>/`).

**STROGO van obima → kasnije priče/Epici (vidi Dev Notes — Granica):**
- **Email notifikacija / auto-reply / pun anti-spam (rate-limit/CAPTCHA)** = **Story 5.2** (aditivno, IR #2). 3.2 SAMO snima `Inquiry` u bazu + prikazuje success poruku. **NEMA slanja email-a.** Anti-spam u 3.2 je SAMO light **honeypot** (skriveno polje) — `django-ratelimit`/CAPTCHA su 5.2.
- **`{% loc %}` template tag** = **Epik 6** (tag NE postoji). 3.2 renderuje opis preko `description_sr` direktno (SR MVP), kao 2.2/3.1. FR12 referencira `{% loc %}` ali se tag odlaže.
- **Slične nekretnine (FR16)** — epics 3.2 ima i AC za max 3 curated „slične nekretnine" na dnu. **U 3.2 OPCIONO/minimalno** (vidi AC7): ako se radi, prost queryset `signature+active`, isključi tekuću, `[:3]`, reuse `property-card` markup; **NIJE kritičan put** i ne sme rušiti stranicu ako je prazan. (Glavni fokus priče = galerija + sidebar + features + mapa + agent forma.)
- **Pun i18n** (`i18n_patterns`, `set_language`, `.po`/`.mo`, `{% loc %}`, EN detalj) = **Epik 6**. Ovde: UI stringovi kroz `{% trans %}`; sadržaj iz baze kroz `_sr` polja / display metode.
- **WebP/responsive `srcset`, SEO meta/Schema.org, perf** = Epik 6. 3.2 koristi `hero_image.url`/`image.url` direktno + placeholder fallback (2.1 obrazac).
- **base.html okvir** (header/footer/lang-switcher/kaskada) = 2.1. 3.2 NE menja header/nav; samo `{% extends %}` i puni `content`/`extra_css`/`extra_js`.
