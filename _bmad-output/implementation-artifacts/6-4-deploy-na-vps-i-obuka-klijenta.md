# Story 6.4: Deploy na VPS i obuka klijenta — Hetzner Ubuntu LTS (venv + Gunicorn/systemd + Nginx), PostgreSQL 16, Let's Encrypt, backup, produkcioni rate-limit, primopredaja klijentu

## Opis

As a vlasnik sajta,
I want sajt u produkciji na VPS-u sa HTTPS, backup-om i obukom,
So that sajt radi pouzdano i samostalno ga održavam kroz CMS.

Ovo je **POSLEDNJA priča Epika 6 (Faza 6 + 7) i poslednja priča MVP-a** — sajt iz lokalnog razvoja odlazi u **produkciju na čist Hetzner Cloud VPS (Ubuntu LTS)**, dobija **HTTPS, automatski backup, produkcioni rate-limit** i završava **primopredajom + obukom klijenta** za samostalan rad u CMS-u. Realizuje **NFR-5 (Sigurnost)** u celosti (HTTPS, CSRF, rate-limit, admin path, kredencijali u env-u, redovan backup) i zaključava deploy okruženje u kojem se **NFR-1 (performanse)** stvarno meri (Lighthouse procedura iz Story 6.3 T6).

Trenutno stanje (nasleđeno — POTVRĐENO čitanjem koda):
- **`config/settings/prod.py` POSTOJI od Story 1.1** (POTVRĐENO): `DEBUG=False` (l.4), `ALLOWED_HOSTS=env("ALLOWED_HOSTS")` (l.6), `SECURE_SSL_REDIRECT`/`SESSION_COOKIE_SECURE`/`CSRF_COOKIE_SECURE=True` (l.11–13), HSTS 1god+subdomains+preload (l.15–17), `SECURE_CONTENT_TYPE_NOSNIFF` (l.19), **`SECURE_PROXY_SSL_HEADER=("HTTP_X_FORWARDED_PROTO","https")` (l.21 — KLJUČNO: Django prepoznaje HTTPS iza Nginx TLS terminacije)**, Mailgun anymail backend fail-fast iz env-a (l.27–36). **prod.py NEMA:** produkcioni `CACHES` override (deljeni cache), X-Forwarded-For IP konfiguraciju za rate-limit, ukidanje `SILENCED_SYSTEM_CHECKS`, odluku o statičkom serviranju, send-timeout za Mailgun (l.37–40 TRACKED kao 6.4) — **6.4 ih dodaje/rešava**.
- **`base.py` rate-limit je MVP jedno-procesni (POTVRĐENO):** `CACHES["default"]=LocMemCache` (l.148–152) + `SILENCED_SYSTEM_CHECKS=["django_ratelimit.E003","django_ratelimit.W001"]` (l.157–160). Komentar (l.142–156) **EKSPLICITNO upućuje na Story 6.4:** prod sa više worker-a → deljeni cache (Redis/Memcached) + stvarni klijentski IP iza Nginx-a (X-Forwarded-For) → **obara potrebu za utišavanjem**. Ovo je tačno AC6 (epics 6.4 l.560).
- **`base.py` static/media (POTVRĐENO):** `STATIC_URL="static/"` (l.212), `STATIC_ROOT=BASE_DIR/"staticfiles"` (l.214), `MEDIA_URL="/media/"` (l.219), `MEDIA_ROOT=BASE_DIR/"media"` (l.220). **MIDDLEWARE (l.90–104) NEMA WhiteNoise** iako ga arh §4 (l.190) predviđa kao „sigurnu rezervu" — **odluka ZAKLJUČANA: Nginx-only static (MVP default); `whitenoise` OSTAJE instaliran u base.txt (asertovan testom — ne uklanjati), ali se NE dodaje u MIDDLEWARE** (vidi Dev Notes — WhiteNoise).
- **Storage je storage-agnostic ali S3 NIJE aktiviran (POTVRĐENO, iz 6.3):** `STORAGES["default"]` bira backend preko `core.storages.default_storage_backend(STORAGE_BACKEND)` (l.233–242); `STORAGE_BACKEND` default `"local"` → `FileSystemStorage`; `boto3` komentarisan u base.txt. **MVP deploy default = lokalni FS + Nginx servira `/media/`** (arh §1.4 l.94); S3 aktivacija (boto3+AWS_*+`STORAGE_BACKEND=s3`) je OPCIONA odluka u 6.4 (vidi Dev Notes — Granica).
- **`requirements/prod.txt` (POTVRĐENO):** `-r base.txt` + `gunicorn`. `psycopg[binary]` (PostgreSQL adapter) je u base.txt (arh §4). 6.4 dodaje produkcione runtime pakete SAMO ako se odluči (deljeni cache klijent / ipware).
- **PostgreSQL je PRVI PUT u stvarnoj upotrebi (KRITIČNO — vidi Dev Notes):** lokalni razvoj koristi SQLite (`dev-local.sqlite3`) i test koristi SQLite `:memory:`; **lokalno NEMA PostgreSQL-a**. `DATABASE_URL` (`base.py` l.136) je već PostgreSQL-ready (`env.db()`), ali migracije **nikad nisu izvršene protiv prave PostgreSQL 16 baze**. VPS je PRVO mesto gde Postgres stvarno radi → **verifikacija `migrate` protiv PostgreSQL 16 je STVARAN rizik/zadatak, ne pretpostavka**.

**Obim ove priče = DEPLOY + PRODUKCIONIZACIJA + PRIMOPREDAJA (NFR-5 / arh §3/§6/§7 / PRD Faza 7):**
1. **Provisioning VPS-a (Hetzner Ubuntu LTS, čist server):** non-root app korisnik, **Python 3.13** (deadsnakes PPA ili source build — NIJE default na Ubuntu LTS-u; vidi Dev Notes), PostgreSQL 16, Nginx, system paketi.
2. **Deploy aplikacije:** git clone → venv (Python 3.13) → `pip install -r requirements/prod.txt` → `.env` (chmod 600, vlasnik app korisnik) → **`migrate` protiv PostgreSQL 16 (verifikovati!)** → `collectstatic`.
3. **Gunicorn (systemd):** `gunicorn.service` (+ socket) na Unix socket-u, `EnvironmentFile=`, auto-restart.
4. **Nginx (reverse proxy):** `location /static/` + `location /media/` direktno sa diska; sve ostalo proxy na Gunicorn socket; gzip, cache headeri, security headeri; **`proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` + `X-Forwarded-Proto`**.
5. **TLS (Certbot/Let's Encrypt):** sertifikat + HTTP→HTTPS redirect + auto-renew (systemd timer). HSTS već u `prod.py`.
6. **Produkcioni rate-limit (AC6 — NFR-5):** deljeni cache (Redis/Memcached) za `default` ILI namenski `ratelimit` alias + stvarni klijentski IP iza Nginx-a (X-Forwarded-For) → ukloniti `SILENCED_SYSTEM_CHECKS` u prod-u.
7. **Backup:** dnevni `pg_dump` (cron, retencija 7 dnevnih + 4 nedeljna) + offsite kopija; dnevni `tar`/`rsync` `media/`.
8. **Mailgun:** verifikacija domena (SPF/DKIM/MX) pre prvog prod slanja; dodati send-timeout (prod.py TRACKED l.37–40).
9. **Primopredaja + obuka klijenta (Faza 7):** klijentska dokumentacija (kako se koristi Velegrad CMS admin: nekretnine, slike/reorder, upiti, SiteSettings, dvojezičan unos, preview/objava) + osnovne operativne smernice (backup/restore, gde su logovi, kako se obnavlja sertifikat).

**STROGO van obima → ostale priče / odloženo (vidi Dev Notes — Granica):**
- **Docker / docker-compose** = NE za MVP (arh §7 LOCKED; direktan venv+systemd+Nginx). Docker put je dokumentovan u arh §7 ali ODLOŽEN.
- **AJAX `/api/` listing sloj, virtual tour, Facebook Pixel, FR lokalizacija** = van MVP-a (arh §8; nisu deploy briga).
- **CDN ispred media/static (CloudFront i sl.)** = van MVP-a (Nginx servira direktno; CDN je buduća optimizacija — 6.3 Dev Agent Record S3 napomena).
- **Funkcionalne izmene aplikativnog koda** = NE — 6.4 ne menja business logiku; samo produkcionizuje settings/infra + dokumentaciju (izuzev scoped produkcionizacije rate-limita i Mailgun timeout-a, koji su deploy-uslovljeni).
- **Migracija na Python 3.12** = samo DOKUMENTOVANI fallback ako 3.13 napravi problem (npr. wheel nedostupan za neki paket) — primarni cilj je 3.13 (vidi Dev Notes).
- **Celery / async slanje email-a (broker, worker)** = EKSPLICITNO VAN OBIMA MVP-a — nema brokera ni u arhitekturi ni u requirements. Mailgun send-timeout se rešava ISKLJUČIVO custom `requests` session-om sa timeout-om (AC4/T7, vidi Dev Notes — Mailgun). NE graditi async/Celery stek u 6.4.
- **WhiteNoise u `MIDDLEWARE`** = van obima 6.4 — `whitenoise` OSTAJE instaliran u base.txt (asertovan testom, ne uklanjati), ali se NE ožičava u MIDDLEWARE; static servira Nginx (AC1, vidi Dev Notes — WhiteNoise). Dodavanje u MIDDLEWARE je zasebna van-obima odluka.
